# Context

A few documents worth reading if you want to understand why Etak is shaped the way it is.

- [**About the name**](about-the-name.md) — the Polynesian navigation concept the tool is built around. Short. Start here.
- [**Foundations**](foundations.md) — the research the discovery plugin sits on. Longer. Read it when you're curious why a specific design choice was made.
- [**Develop foundations**](develop-foundations.md) — the companion for the develop plugin. Why `etak-develop` looks different from a generic AI coding assistant.
- [**Documentation style guide**](documentation-style-guide.md) — voice and register for anything written in this repository.

These pages are not required reading. The [quick start](../quickstart.md) and [tutorials](../tutorials/) will get you productive without them. But the philosophy is not decorative. If the tool feels subtly different from others you've used, these documents are why.
