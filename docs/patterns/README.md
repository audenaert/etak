# Patterns

Practical guidance for people who have been using Etak for a few weeks and have started to feel the edges. This is not a walkthrough. It assumes you know the moves (orient, explore, sound, prioritize, experiment, plan, spec, build) and are now wondering how to use them well when the graph has gotten noisy, the tool has proposed something you don't believe, or you're trying to work with another person across the discovery-to-develop seam.

Come here with a question, not to read straight through. Each page is a short piece of advice on one recognizable situation. Skip to the one that matches what you're hitting.

## The patterns

- [**Graph hygiene**](graph-hygiene.md): what to do when the graph has accumulated stale experiments, duplicate opportunities, and ideas nobody remembers adding.
- [**Pushing back on the tool**](pushing-back.md): the tool proposes an assumption you don't hold, or cites evidence that doesn't exist. How to challenge it without losing the thread.
- [**PM and engineer together**](pm-engineer-partnership.md): using discovery and develop across a seam, what the `from_discovery` link actually does, who owns which decisions.
- [**Skill or agent**](agent-vs-skill.md): when to pair interactively, when to dispatch and review. The cost and control tradeoff, and how to review agent output so it earns the autonomy.
- [**Anti-patterns**](anti-patterns.md): five habits that quietly erode the value of the graph, and what to do instead.

## A note on register

These pages are advisory, not pedagogical. They assume you've made some of the mistakes they describe. If something rings a bell, the pattern is probably for you. If nothing rings a bell, you either haven't used the tool long enough or you've already internalized the lesson. Either is fine.
